# 雀魂AIアシスタント PRD

**プロジェクトコード**: jantama-ai
**バージョン**: 1.0 (MVP)
**作成日**: 2026-05-07
**作成者**: Akimasa
**ステータス**: ドラフト

---

## 1. プロジェクト概要

### 1.1 背景

雀魂(Mahjong Soul)で対局中、「自分の手をどう進めるべきか」「いまこの局面で何を切るのが期待値が高いのか」をリアルタイムで判断するのは難しい。プロ級の麻雀AI(Mortal等)は存在するが、雀魂と直接連携するインタフェースは整っておらず、対局中に活用するには手作業で局面を入力する必要がある。

### 1.2 目的

雀魂の画面をリアルタイムにキャプチャし、麻雀AI Mortalによる推論結果(推奨アクションと期待値)を即座に表示するデスクトップアプリを開発する。これにより、対局中の打牌判断を支援し、上達の手助けとする。

### 1.3 スコープサマリー

| 区分 | 内容 |
|---|---|
| **やる** | 雀魂・四人麻でのリアルタイム打牌アクション提案 |
| **やらない** | 自動操作、棋譜振り返り、成績管理、三人麻、雀魂以外のサイト対応 |

---

## 2. ペルソナとゴール

### 2.1 ペルソナ

- **想定ユーザー**: 開発者本人(Akimasa)1人
- **環境**: Windows PC、AMD Ryzen AI 300シリーズ + Radeon 860M内蔵GPU
- **利用シーン**: 雀魂で対局中、思考時間内に推奨手を確認したい

### 2.2 解決したい課題

雀魂対局中に「他の上級者ならこの局面で何を切るか」をリアルタイムで知ることができない。

### 2.3 成功の定義

- 自分の手番が来た際に、Mortalの推奨アクションと上位候補の期待値スコアが表示されること
- MVP段階では認識から表示まで3秒以内、将来的には1秒以内を達成すること

---

## 3. ユーザーストーリー(ジャーニー)

### 3.1 メインジャーニー

1. ユーザーが雀魂(Steam版またはブラウザ版)を起動
2. 本アプリ(雀魂AIアシスタント)を起動
3. 設定画面でキャプチャ対象ウィンドウとMortalモデルパスを指定(初回のみ)
4. メイン画面の「監視ON」をクリック
5. 雀魂で対局を開始
6. 自分の手番が来ると、メイン画面に推奨アクションと期待値スコアが表示される
7. ユーザーは表示を見て、雀魂上で手動で打牌する
8. 対局終了後、「監視OFF」をクリック

### 3.2 状態遷移

メイン画面は以下の状態を内部で切り替えて表示する。

| 状態 | 説明 |
|---|---|
| 未設定 | 初回起動・設定未完了 |
| 待機中 | 設定済み・監視OFF |
| 監視中・盤面なし | 監視ON・対局していない/手番でない |
| 監視中・推奨表示中 | 自分の手番・推奨アクション表示中 |
| エラー | 認識失敗・推論失敗・ウィンドウ消失等 |

## 4. 機能要件

### 4.1 Must(MVP必須)

| ID | 機能 | 説明 |
|---|---|---|
| F-01 | キャプチャ対象選択 | 起動中のウィンドウ一覧から雀魂のウィンドウを選択する |
| F-02 | Mortalモデルパス指定 | Mortalモデルファイル(.pth等)のローカルパスを指定する |
| F-03 | 監視ON/OFF切り替え | メイン画面のボタンで画面監視を開始/停止する |
| F-04 | 画面キャプチャ | 指定ウィンドウを定期的にキャプチャする |
| F-05 | 盤面認識 | 手牌・河・ドラ・点棒・場況・副露を認識する |
| F-06 | 天鳳JSON変換 | 認識結果をMortal入力形式(天鳳JSON互換)に変換する |
| F-07 | Mortal推論実行 | 天鳳JSONを入力に推論を実行する |
| F-08 | 推奨アクション表示 | 推奨打牌(上位候補)+ 各候補の期待値スコアを表示する |
| F-09 | 打牌以外の推奨 | 鳴き(チー/ポン/カン)・リーチ・和了(ロン/ツモ)の推奨も表示する |
| F-10 | エラー状態表示 | 認識失敗・推論失敗・ウィンドウ消失時に状態を表示する |
| F-11 | 推論履歴の自動削除 | 30日経過した推論履歴は自動削除する |
| F-12 | 牌画像の自動削除 | 7日経過した画像サンプルは自動削除する |
| F-13 | エラーログの自動削除 | 90日経過したエラーログは自動削除する |

### 4.2 Should(できれば入れたい)

| ID | 機能 | 説明 |
|---|---|---|
| S-01 | 推奨理由の説明 | LLM(Claude API等)で「この牌を切る理由」を自然言語で生成する |
| S-02 | 危険牌表示 | 他家の捨牌から危険牌を判定し、ベタ降り時の安全牌を表示する |
| S-03 | デバッグビュー | 認識した手牌・河の認識結果をUIで確認できる |
| S-04 | 設定画面拡張 | 表示位置・不透明度・ホットキーをUIから変更できる |

### 4.3 Could(余裕があれば)

| ID | 機能 | 説明 |
|---|---|---|
| C-01 | オーバーレイモード | 雀魂画面の上に半透明で重ねて表示する |
| C-02 | 認識結果の手動修正 | 認識ミスをユーザーが修正できる |

### 4.4 Won't(やらない・スコープ外)

- 雀魂の自動操作(ボット化、自動打牌)
- 対局後の棋譜振り返り機能
- 成績管理・長期的な上達記録表示
- 雀魂以外の麻雀サイト対応(天鳳、MJ等)
- 三人麻対応
- 雀魂のメモリ・通信解析

### 4.5 権限・ロール

認証なし。アプリを起動できる人=全機能を使える。マルチユーザー想定なし。

## 5. 画面要件

### 5.1 画面一覧

| 画面ID | 画面名 | 概要 |
|---|---|---|
| SCR-01 | メイン画面 | 監視ON/OFF、推奨アクション表示、各種ステータス |
| SCR-02 | 設定画面 | キャプチャ対象選択、Mortalモデルパス指定 |

### 5.2 メイン画面(SCR-01)

#### レイアウト方針

「推奨アクションを中央に大きく」配置し、ステータス・補足情報は上下に小さく配置する。一目で打牌候補が分かることを最優先する。

#### 構成要素

```
┌─────────────────────────────────────────────┐
│ [上部] ステータスバー                          │
│  - 監視状態(ON/OFF)                           │
│  - キャプチャ対象ウィンドウ名                   │
│  - 最終認識時刻                                │
│  - [設定]ボタン                                │
├─────────────────────────────────────────────┤
│                                             │
│ [中央] 推奨アクション(大)                      │
│                                             │
│  推奨: 6m を切る                              │
│                                             │
│  候補:                                        │
│  1. 6m  期待値: +0.32                         │
│  2. 9p  期待値: +0.18                         │
│  3. 1z  期待値: -0.05                         │
│                                             │
│  アクション種別: [打牌]                        │
│                                             │
├─────────────────────────────────────────────┤
│ [下部] 認識結果サマリー(小)                    │
│  手牌: 1m 2m 3m 4m 5m 6m 7p 8p 9p 1z 2z 3z 5m │
│  自風: 東 / 場風: 東 / 巡目: 6 / ドラ: 5p     │
└─────────────────────────────────────────────┘
[監視OFF / ON]ボタン
```

#### 状態別の表示

| 状態 | 中央エリアの表示 |
|---|---|
| 未設定 | 「設定画面で初期設定を行ってください」+ [設定画面へ] ボタン |
| 待機中 | 「監視を開始してください」+ [監視ON] ボタン |
| 監視中・盤面なし | 「対局を待機中...」+ ローディング表示 |
| 監視中・推奨表示中 | 推奨アクション・候補・期待値を表示 |
| エラー | エラーメッセージ + 再試行/設定への誘導 |

### 5.3 設定画面(SCR-02)

#### 構成要素

- **キャプチャ対象ウィンドウ選択**: 起動中のウィンドウ一覧をドロップダウンで表示
- **Mortalモデルパス**: ファイル選択ダイアログ(`.pth` 等)
- **データ保存期間**(MVP範囲外、Should): 推論履歴/画像/ログの保存日数設定
- **保存ボタン / メイン画面に戻るボタン**

### 5.4 画面遷移

```
[起動]
   │
   ▼
[メイン画面]──────────────────────────┐
   │                                 │
   │ [設定]ボタン                     │
   ▼                                 │
[設定画面]                            │
   │                                 │
   │ [保存] / [戻る]                  │
   └─────────────────────────────────┘
```

メイン画面と設定画面の2画面のみ。それ以外の状態変化はメイン画面内で表示切り替えする。

## 6. データモデル

### 6.1 エンティティ一覧

| エンティティ | 用途 | 保存先 |
|---|---|---|
| AppSettings | アプリ設定 | 設定ファイル(JSON/TOML) |
| GameState | 現在の盤面状態 | メモリのみ(永続化なし) |
| InferenceLog | 推論履歴 | SQLite |
| TileImageSample | 牌画像サンプル | ファイル+SQLiteメタデータ |
| ErrorLog | エラーログ | SQLite |

### 6.2 主な属性

#### AppSettings(設定ファイル)

- `capture_target_window_id`: キャプチャ対象ウィンドウのID/タイトル
- `mortal_model_path`: Mortalモデルファイルのパス
- `window_position`, `window_size`: アプリウィンドウの位置・サイズ
- `data_retention_days`: 保存期間(推論履歴30/画像7/ログ90)
- `hotkey_settings`: ホットキー設定(Should対応)

#### GameState(メモリのみ)

天鳳JSON互換フォーマット。手牌、河、ドラ、点棒、巡目、自風、場風、副露を含む。

#### InferenceLog(SQLite)

- `id`, `timestamp`, `game_state_snapshot` (JSON)
- `mortal_output` (JSON: 推奨候補と期待値)
- `recommended_action_type`(打牌/リーチ/鳴き/和了)

#### TileImageSample(ファイル+SQLite)

- `id`, `timestamp`, `image_file_path`
- `recognition_result`, `confidence_score`

#### ErrorLog(SQLite)

- `id`, `timestamp`, `error_type`, `message`, `stack_trace`
- `related_game_state` (JSON、任意)

### 6.3 保存ポリシー

| データ | 保存期間 |
|---|---|
| 推論履歴 | 30日 |
| 牌画像サンプル | 7日 |
| エラーログ | 90日 |
| アプリ設定 | 永続(削除なし) |

期限経過分はバックグラウンドジョブで自動削除する。

## 7. 非機能要件

### 7.1 パフォーマンス

| 項目 | MVP目標 | 将来目標 |
|---|---|---|
| 認識から表示までの総時間 | 3秒以内 | 1秒以内 |
| 内訳:画面キャプチャ | ~50ms | ~50ms |
| 内訳:牌・場況認識 | ~500ms | ~200ms |
| 内訳:天鳳JSON変換 | ~10ms | ~10ms |
| 内訳:Mortal推論 | ~1.5s | ~500ms |
| 内訳:UI描画 | ~100ms | ~50ms |

### 7.2 動作環境

- **OS**: Windows 11 のみサポート
- **CPU/GPU**: AMD Ryzen AI 300シリーズ + Radeon 860M内蔵GPU
- **推論実行環境**: PyTorch on Windows + ROCm 7.2.1以降(public preview段階)
- **フォールバック**: ROCmが不安定な場合はCPU推論に切替

### 7.3 セキュリティ・プライバシー

- 雀魂のメモリ・通信は一切読まない(画面キャプチャのみ)
- 雀魂の自動操作は一切しない
- データはすべてローカル保存(クラウド送信なし)
- 認証機能なし(個人ローカルツールのため)

### 7.4 信頼性・回復性

| シナリオ | 振る舞い |
|---|---|
| 認識失敗 | スキップして次フレームへ。ログに記録 |
| Mortal推論失敗 | エラー表示+ログ記録+監視は継続 |
| 雀魂ウィンドウ消失 | 「キャプチャ対象が見つかりません」表示+定期再検出 |
| アプリクラッシュ | 復元なし(再起動でリセット) |

### 7.5 監視・ログ

- エラーログはSQLiteに記録(90日保持)
- 推論履歴は将来の認識精度改善・自分の打牌傾向分析に活用可能(30日保持)

## 8. 技術スタック

### 8.1 全体構成図

```
┌────────────────────────────────────────────────────────┐
│ Tauri Desktop App (Windows .exe)                       │
│                                                        │
│  ┌──────────────────────┐  ┌──────────────────────┐   │
│  │ Frontend (WebView)   │  │ Backend (Rust)       │   │
│  │ - React + TS + Vite  │◄─┤ - 画面キャプチャ      │   │
│  │ - Tailwind v4        │  │ - サブプロセス管理    │   │
│  │ - shadcn/ui          │  │ - SQLite              │   │
│  └──────────────────────┘  └──────┬───────────────┘   │
│                                   │ stdin/stdout      │
│                                   │ (JSON-lines)      │
│             ┌─────────────────────┴────────────────┐  │
│             │                                      │  │
│             ▼                                      ▼  │
│  ┌──────────────────┐               ┌──────────────────┐
│  │ 認識プロセス        │               │ Mortalプロセス     │
│  │ (Python+OpenCV)  │──天鳳JSON─►  │ (Python+PyTorch) │
│  │ 画像→盤面構造化    │               │ 推論→推奨候補       │
│  └──────────────────┘               └──────────────────┘
└────────────────────────────────────────────────────────┘
                          │
                          ▼
                    [雀魂ウィンドウ]
                    (Steam版 or ブラウザ版)
```

### 8.2 採用技術

#### フロントエンド(WebView)

- **React** + **TypeScript** + **Vite**
- **Tailwind CSS v4**
- **shadcn/ui**
- 状態管理: 外部ライブラリなし(useState/useReducer)

#### Tauri(Rust側)

- **Tauri 2.x**
- 画面キャプチャ: `xcap` クレート
- Pythonサブプロセス管理: 標準ライブラリの`std::process::Command` + 標準入出力
- DB: `tauri-plugin-sql` (SQLite)
- 設定ファイル: `tauri-plugin-store` または独自JSON/TOML

#### Python側(2プロセス構成)

##### 認識プロセス

- Python 3.12
- OpenCV(画像処理・テンプレートマッチング)
- PyTorch(軽量CNN、必要に応じて)
- 入力:画像バイト列(stdin)、出力:天鳳JSON(stdout)

##### Mortalプロセス

- Python 3.12
- PyTorch (ROCm 7.2.1+ on Windows / フォールバックCPU)
- Mortal本体(GitHub: Equim-chan/Mortal)
- 入力:天鳳JSON(stdin)、出力:推奨候補+期待値JSON(stdout)

#### ストレージ

- **SQLite**(`tauri-plugin-sql`)
- ローカルファイルシステム(画像サンプル)
- JSON/TOMLファイル(アプリ設定)

#### パッケージング

- Tauriバンドル(.exe / .msi)
- PyInstallerでPython側をexe化してTauriバンドルに同梱
- Mortalモデルファイルも同梱(または初回起動時ダウンロード)

#### 開発ツール

- **リポジトリ**: GitHubプライベート
- **フロント**: ESLint + Prettier + Vitest
- **Python**: Ruff + Black + pytest
- **CI**: GitHub Actions(lint/test)

### 8.3 雀魂対応範囲

- Steam版クライアント・ブラウザ版の両方
- ウィンドウタイトルまたはプロセス名で対象を特定する方式

## 9. リスクと対策

| リスク | 影響度 | 対策 |
|---|---|---|
| ROCm on Windowsがpublic preview段階で不安定 | 高 | CPU推論フォールバックを実装。CPU推論でも3秒以内なら許容 |
| 雀魂のUI変更で認識が壊れる | 中 | テンプレートマッチングを設定ファイル化し、再キャリブレーションを容易に |
| 雀魂規約上のグレーゾーン | 中 | 自動操作・メモリ読み取り・通信解析は一切しない方針を厳守 |
| Mortalモデルのライセンス | 低 | 個人利用に留める。配布しない |
| 思考時間内に推論が間に合わない | 中 | MVPは3秒以内に緩和。間に合わない場合はスキップして次フレームに進む |

## 10. 開発計画(ロードマップ案)

期限なしの個人プロジェクトのため、ストーリーポイント順の実装目安を示す。

### Phase A: 基盤構築

- Tauri + React + TypeScript の雛形作成
- 画面キャプチャ(Rust側)の実装
- メイン画面・設定画面の骨組み

### Phase B: Python連携

- 認識プロセス・Mortalプロセスのスケルトン
- Tauri ↔ Python のJSON-lines通信確立
- 設定画面からモデルパスを指定し、Mortalが起動できる状態

### Phase C: 認識実装

- 雀魂のスクリーンショットから手牌・河を認識
- 天鳳JSONへの変換

### Phase D: 推論・表示

- Mortal推論結果のUI表示
- 状態管理(未設定/待機/監視中/エラー)

### Phase E: データ永続化

- SQLite統合
- 推論履歴・画像サンプル・エラーログの記録
- 自動削除ジョブ

### Phase F: パッケージング

- PyInstallerでPythonをexe化
- Tauriバンドルに同梱
- インストーラ生成

### Phase G(Should): LLM連携

- 推奨理由の自然言語生成
